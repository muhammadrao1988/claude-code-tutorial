# Specs

This folder contains spec documents for each completed tutorial.

## Purpose

Specs record what was built, when, and which files were created. They serve as a log of completed work and make it easy to review what each tutorial covers.

## Format

Each spec file is named `[number]-[topic].md` and includes:
- Tutorial number and title
- Completion date
- List of deliverables
- List of files created
- Examples covered

## Creating Specs

Specs are created automatically by the spec-tracker agent when you run `/publish`. You should not need to create them manually.
