# Try it yourself: Memory (CLAUDE.md)

## Exercise

Create a CLAUDE.md that teaches Claude about YOU and YOUR work.

### Step 1: Pick a folder

Choose any folder where you use (or want to use) Claude Code. It could be a project, a notes folder, or even your home directory.

### Step 2: Create the file

```bash
cd /path/to/your/folder
touch CLAUDE.md
```

### Step 3: Write your instructions

Open CLAUDE.md and answer these questions in bullet points:

- What is this project or folder about?
- What tools or tech do you use?
- What rules should Claude always follow here?
- What commands do you run often?

### Step 4: Test it

Start a new Claude Code session in that folder:

```bash
claude
```

Ask Claude: "What do you know about this project?"

It should repeat back the key details from your CLAUDE.md. If it doesn't, your file might not be in the right location.

### Step 5: Refine

Use Claude for a few tasks. When it does something you don't like, add a rule to CLAUDE.md to prevent it next time. Your CLAUDE.md gets better the more you use it.
