## Tutorial Project Rules

- Maximum 2 tutorials published per day
- Every tutorial must have both a non-tech and developer example
- No emojis in tutorial content
- All code blocks must be copy-pasteable
- Keep tutorial READMEs under 200 lines
- Use the tutorial-style skill when writing content
- Use the spec-tracker agent after completing a tutorial
- Always check the backlog before starting new work
- Commit often with descriptive messages
